# Hosting GitHub Pages

## Recommended Setup

Use the `docs/` folder as the Pages source so the site can be published directly from the repository.

## Steps

1. Push this repository to GitHub.
2. Make sure `docs/index.md` exists and is the landing page.
3. Open the repository on GitHub and go to Settings.
4. Select Pages.
5. Under Build and deployment, choose Deploy from a branch.
6. Pick the `main` branch and the `/docs` folder.
7. Save the settings and wait for GitHub to build the site.
8. Open the published Pages URL shown in the settings page.

## Optional Cleanup

If you want the site to feel more complete, you can add a custom theme, a navigation sidebar, or a dedicated `README.md` link list that points into the published Pages content.
