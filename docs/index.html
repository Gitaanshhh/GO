Go fundamentals

Overview

Why Go?

Go was designed for modern systems and offers advantages that make it well-suited for backend and cloud development.

Designed for

  • multicore systems
  • cloud infrastructure
  • distributed systems
  • backend/network services

Key advantages

  • built-in concurrency
  • lightweight threading via goroutines
  • fast compile + execution
  • simple syntax
  • statically typed
  • compiled to a single binary

Combines Python-like simplicity with C/C++-like performance.

1. Essentials

Basics

Variables, constants, zero values, and pointers are the foundation for the rest of the notes.

Variables and constants

var x int = 5
var y = 5
z := 5

var a, b int = 1, 2

const Pi = 3.14

const (
  A = iota
  B
  C
)

Types and zero values

var i int
var f float64
var b bool
var s string
var p *int

Pointers

x := 42
p := &x
*p = 100

ps := &Person{Name: "Ali"}
ps.Name

p2 := new(int)

The short declaration form is common in functions, and unused variables must be removed before compilation.

2. Collections

Arrays and Slices

Arrays have fixed size; slices are flexible views over backing arrays.

Arrays

var a [3]int
b := [3]int{1, 2, 3}
c := [...]int{1, 2, 3}

x := b
x[0] = 99

Assignments copy the full array value, so mutations on the copy do not affect the original.

Slices and range

s := []int{1, 2, 3}
s2 := make([]int, 5)
s3 := make([]int, 3, 5)

s = append(s, 4, 5)

sub := s[1:4]
dst := make([]int, len(s))
copy(dst, s)

for i, v := range s {
  _ = i
  _ = v
}

Sub-slices share storage unless you copy the data into a fresh slice.

3. Lookup

Maps

Maps are the standard Go hash table and are ideal when the key matters more than ordering.

Map basics

m := map[string]int{"a": 1, "b": 2}
m2 := make(map[string]int)

m["key"] = 42
val := m["key"]
val, ok := m["key"]
delete(m, "key")

Use the two-value lookup when you need to tell the difference between a missing key and a zero value.

Map of slices

graph := make(map[string][]string)
graph["a"] = append(graph["a"], "b", "c")

4. Branching

Control Flow

Go keeps loops and branching small and explicit, which makes the flow easy to scan.

For loops

for i := 0; i < 5; i++ {}

for x < 100 {
  x *= 2
}

for {
  break
}

If and switch

if err := doThing(); err != nil {
  log.Fatal(err)
}

switch x {
case 1:
  fmt.Println("one")
case 2, 3:
  fmt.Println("two|three")
default:
}

Use fallthrough explicitly when you want C-style switch behavior.

5. Reuse

Functions

Function signatures, variadics, closures, and deferred cleanup are core Go patterns.

Signatures and returns

func add(a, b int) int { return a + b }

func divide(a, b float64) (float64, error) {
  if b == 0 { return 0, errors.New("div by zero") }
  return a / b, nil
}

Variadic functions and closures

func sum(nums ...int) int {
  total := 0
  for _, n := range nums {
    total += n
  }
  return total
}

counter := func() func() int {
  n := 0
  return func() int {
    n++
    return n
  }
}

Defer

defer f.Close()

defer is the standard cleanup mechanism and runs in last-in, first-out order.

6. Data models

Structs

Structs, methods, embedding, and interfaces are how Go models behavior and composition.

Structs and methods

type Person struct {
  Name string
  age  int
}

func (p Person) Greet() string {
  return "Hi, " + p.Name
}

func (p *Person) Birthday() {
  p.age++
}

Embedding and interfaces

type Employee struct {
  Person
  Company string
}

e := Employee{
  Person:  Person{Name: "Ali", age: 25},
  Company: "Cisco",
}

type Greeter interface {
  Greet() string
}

var g Greeter = Person{Name: "Alice"}

Go uses structural typing, so a type satisfies an interface when it has the right methods.

7. Project structure

Packages and Scope

Packages organize code, and exported identifiers are controlled by capitalization.

Packages and imports

package main

import (
  "fmt"
  m "math"
  _ "image/png"
)

Visibility and init

type PublicStruct struct {}
type privateStruct struct {}

func PublicFunc() {}
func privateFunc() {}

func init() {}

Capitalized names are exported, lowercase names stay package-private, and init() runs before main().

8. Parallel work

Concurrency

Goroutines, channels, selects, and mutexes are the main primitives for concurrent Go code.

Go uses a green thread - Abstraction of an actual thread, managed by go runtime. It is cheaper and more lightweight. We only interact with these high level go routines.

Goroutines and wait groups

go func() {
  fmt.Println("concurrent!")
}()

var wg sync.WaitGroup
for i := 0; i < 5; i++ {
  wg.Add(1)
  go func(n int) {
    defer wg.Done()
    work(n)
  }(i)
}
wg.Wait()

Channels, select, and mutex

ch := make(chan int)
bch := make(chan int, 5)

ch <- 42
v := <-ch
close(ch)

select {
case v := <-ch1:
  fmt.Println(v)
case <-time.After(time.Second):
  fmt.Println("timeout")
default:
}

var mu sync.Mutex
mu.Lock()
sharedCounter++
mu.Unlock()

9. Failure paths

Errors

Error handling stays explicit, and panic plus recover should stay reserved for exceptional cases.

Error handling

val, err := strconv.Atoi("42")
if err != nil {
  return fmt.Errorf("parse failed: %w", err)
}

err1 := errors.New("base error")
err2 := fmt.Errorf("context: %w", err1)

Custom errors, panic, and recover

type AppError struct {
  Code int
  Msg  string
}

func (e *AppError) Error() string {
  return fmt.Sprintf("[%d] %s", e.Code, e.Msg)
}

panic("unrecoverable!")

defer func() {
  if r := recover(); r != nil {
    fmt.Println("recovered:", r)
  }
}()

Use panic and recover only for unexpected failures, not as regular control flow.
